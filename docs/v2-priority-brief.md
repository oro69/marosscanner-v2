# MarosScanner v2 Priority Brief

## Core product goal
MarosScanner v2 must feel like a premium consumer-fintech trading app built for fast, confident decisions, not like a patched version of v1.

## Product promise
The app helps the user find, understand, and act on high-conviction setups with real data, clear scoring, strong risk framing, and better execution confidence.

## What v2 must do better than v1
- Cleaner hierarchy
- Faster scanability
- Better mobile flow
- Better position and execution clarity
- Less clutter
- More premium visual quality
- Stronger confidence in trade status and system state

## Build priority order
1. App shell and navigation
2. Swing
3. Picks
4. Live
5. Potential
6. In Play
7. Reconciliation Center
8. More tools

## Core screens for first release
- Swing
- Picks
- Live
- Potential
- In Play

## Screens intentionally delayed
- Full Journal rebuild
- Tax rebuild
- Watchlist expansion
- Deep scanner customization
- Advanced Frozita features

## UX rules
- Mobile first
- Picks opens by default
- Every screen must have one clear primary action
- Every trade card must show risk clearly
- No overloaded cards
- No duplicated labels
- No fake data
- Trade status must be broker-truth aware

## Design direction
- Premium consumer-fintech
- Dark, elegant, modern
- More confidence and clarity
- Less dashboard clutter
- Strong typography and spacing
- Cleaner badges and state labels

## Success test
If a trader opens the app for 10 seconds, they should understand:
1. what the best opportunity is,
2. what the risk is,
3. what state the market is in,
4. and what action they should take next.
